We deployed a Honeypot.

March 14, 2022


Note: In keeping with our vision of spreading security awareness, this article has been written under the assumption that the idea of a honeypot is new to the reader.


No, we didn’t place a huge round pot filled with honey on our cyber defense line.

Honeypots are computer systems intended to simulate a likely target of cyberattacks. They act as a virtual trap to lure attackers by allowing them to exploit vulnerabilities. As a security mechanism, honeypots are often used to study breaches and improve the security policies of the organisation to better respond to real-world attacks.

For analysis on a larger scale, organisations usually create and deploy their own honeypots on the cloud. This broadens the visibility scope of the virtual target, thus inviting a variety and an abundance of exploitation. Security analysts review each of the breaches and suggest improvements to the security policies of the company. Major companies including Panasonic have greatly strengthened the security of their products by using internet honeypots.

On a smaller scale, remote honeypots are an efficient way to study attacks. These are deployed on a remote network instead of the cloud; here, using virtual machines is a safer option as compared to deploying the target on a personal system. For educational purposes, we deployed PenTBox’s remote honeypot on Kali Linux.

Deploying a remote Honeypot using PenTBox


We use the terminal window for this. As we already had PenTBox installed on our Kali box, the installation step has not been shown. To install PenTBox, you can use the command

wget http://sourceforge.net/projects/pentbox18realised/pentbox-1.8.tar.gz

for downloading the files, followed by  

tar xvfz pentbox-1.8.tar.gz

for extracting them.

We first enter the directory containing the PenTBox folder, and use 

./pentbox.rb

to run it.


Honeypots are a part of network tools, so we enter the Network Tools section and choose Honeypot from the menu by entering the appropriate serial number in the prompt. At the introductory level, defaults are always the best, so we go with Fast Auto Configuration.


And voila! Our honeypot has been activated on port 80. Now we shall try breach this vulnerable target. We open our standard web browser,  Mozilla Firefox, and try to access our Kali box by entering

http://<IP_of_our_Kali_box>

We expect an error, and an error is what we get.


Nothing unusual, right? Let’s switch back to our terminal window. Here, what’s interesting to observe is the output displayed now: the honeypot caught the intrusion attempt and provided us with some useful information about the attacker’s system.


Thus, we successfully deployed and tested our remote honeypot. The data returned by the honeypot makes it an incredibly useful security tool and helps stay updated with the latest breaching techniques.


Not every attack can be stopped, and honeypots are a great way to detect and study breaches without the system actually being compromised. In large organisations, honey nets are also being used to monitor more diverse networks in which one honeypot may prove insufficient. These include not only vulnerable servers and systems, but also intentionally susceptible firewalls, routers, and security applications. The international security research organisation Honeynet Project  actively investigates attacks and develops open-source tools to improve internet security.
