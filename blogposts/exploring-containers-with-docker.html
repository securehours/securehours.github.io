Exploring Containers with Docker

July 12, 2022

Which Linux distribution should I use?

This is often the first question that pops into the mind of a user about to install this open-source operating system. Reviews and opinions can either help of confuse users further; besides, only you can decide which distribution suits your comfort. Now, what if there was a way to try out all possible Linux distributions efficiently –
without wasting time and storage bandwidth? Sounds unbelievable? Well, believe it, because Containers are here to help out with this.

We’re going to use Docker to pull various Linux distribution images. First, we need to install docker using the following commands in sequence:

sudo apt install docker.io

sudo snap install docker

sudo docker run hello-world

hello-world is the docker image present on the Docker Hub. On receiving this message as output, we know that our installation is correct and successful.


Now, we can start running our containers. Let’s pull the image of centos first, after which we’ll create a container called first using the image of centos.


That’s it! Our centos container has been created and is running. Let’s go inside and take a look at what we have. To do that, we connect the container to the bash shell.


Notice that the 0e1a43fbc472 in root@0e1a43fbc472 is the container ID of our newly created container. In this way, we can connect to centos and look around in the isolated file system of its own! To return to our terminal, we use exit.

In this way, we can create containers using images of various operating systems. Let’s assume we’ve made a few more containers. Now, we want to view all the running containers we have.


And voila! We have debian, ubuntu, archlinux, alpine and centos all available at the same time, ready to access! As demonstrated above, we can enter any of these containers and use them as required.

Once a container is created, we can start and stop them whenever we want. We can also start and stop multiple containers at the same time.


Suppose we start the container first, which we created just a while earlier. We can view it in the list of running containers. Notice the status of first, which is highlighted below – since we just started it, it has been ‘up’ for a very short time.


However, when we stop first, we observe that it no longer appears in that list.


For exploring more commands to use on containers, one can access the help manual of docker.


Thus, we set up containers using images of various Linux distributions. One can now conveniently explore and compare each option to choose one suitable for them. We can similarly create containers using images of Windows, MacOS and other operating systems. However, keep in mind that containers can only be installed on system based on the same operating system as the containers; Linux-based containers can only be installed on Linux-based systems, and similarly for other operating systems.

Besides this, containers are widely used today due to their consistent operation and increased portability. They are an efficient alternative to virtual machines; while virtual machines virtualize the hardware, containers virtualize the operating system (as we saw above). Besides, containers are much faster to deploy as compared to virtual machines. They are also isolated from other containers, and hence more secure. A whole server can host multiple containers because of their small size (around 10MBs), resulting in considerable economic savings in the industry.
